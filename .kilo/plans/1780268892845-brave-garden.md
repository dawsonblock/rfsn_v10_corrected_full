# Dawson AI Core Implementation Plan

## Goal
Build Dawson AI Core as a self-correcting engineering AI system with RFSN v10 as its runtime optimization layer, including agent orchestration, planning, solving, criticking, judging, and tool execution capabilities.

The main 5 version has resolved critical compilation and import issues:

- ✓ Agent core integration test passes
- ✓ Tools package imports correctly
- ✓ Pyproject.toml includes agent_core and tools
- ✓ Basic KV manager functionality works

- ✓ Agent core components instantiate and interact correctly

## Remaining Critical Issues to Fix

### Phase 1: Fix Telemetry and Runtime Behavior
1. **Fix ClickHouse SQL Construction**
   - Fix malformed SQL: `values = "), (".join(lines)` → `values = ", ".join(lines)`
   - Remove unused MLX import
   - Add execution_mode to ClickHouse table schema (requires DDL update)
   - Consider switching to JSONEachRow for safer inserts

2. **Implement use_compressed_on_miss in Runtime**
   - On cache miss: store KV, then if use_compressed_on_miss=True, retrieve compressed KV to verify
   - Currently constructor has parameter but logic is not implemented

3. **Split Quantization and Sparse Audits in Runtime**
   - Current audit only measures sparse-selection error on same KV
   - Need two separate audits:
     - quantized_dense_vs_original_dense (quantization error vs original)
     - sparse_quantized_vs_quantized_dense (sparsity error on compressed KV)

4. **Fix KV Sign Generation Performance**
   - Replace Python loop over every element with vectorized approach
   - Current hash-based sign generation is correct but slow for large tensors
   - Option: Cache sign masks by (shape, seed, dtype) or implement in Metal

### Phase 2: Fix Remaining Integration Issues
5. **Fix Async Writer Shutdown**
   - Current: stop() sets _stop_event before draining queue
   - Fix: Process queue items before checking shutdown signal to ensure complete drainage

6. **Integrate Adaptive Sparsity Controller**
   - Wire AdaptiveSparsityController into Runtime to dynamically adjust top_k_ratio
   - Currently controller exists but is not used by runtime

7. **Integrate Memory Guard**
   - Wire MemoryGuard into Runtime/KV manager to enforce memory pressure responses
   - Currently exists but not connected to core runtime logic

8. **Fix Bitpack MLX Compatibility**
   - Replace direct mx.bfloat16 reference with version-safe detection
   - Evaluate and fix unpack() slice assignment if MLX incompatible

### Phase 3: Final Quality Improvements
9. **Enhance ToolRunner Security**
   - Add strong workspace containment to prevent sandbox escapes
   - Improve permission matching from substring-based to token-aware

10. **Fix Benchmarks and Reporting**
    - Add value-quality metrics to KV and end-to-end benchmarks
    - Separate cache-miss and cache-hit behavior in end-to-end benchmark
    - Ensure benchmark_harness reports meaningful quality thresholds

11. **Clean Packaging Artifacts**
    - Remove __pycache__/ and *.pyc files from source releases
    - Ensure .kilo/plans/ is only included if intentionally shipping planning artifacts

12. **Update README Accuracy**
    - Remove overclaims: production-grade, fused kernel, guaranteed test pass rates
    - Accurate status: Alpha prototype with verified core functionality
    - Clarify MLX testing requirements and actual validated capabilities
