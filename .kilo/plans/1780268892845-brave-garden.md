# Dawson AI Core Implementation Plan

## Goal
Build Dawson AI Core as a self-correcting engineering AI system with RFSN v10 as its runtime optimization layer, including agent orchestration, planning, solving, criticking, judging, and tool execution capabilities.

## Critical Issues to Fix (Based on Deep Analysis)

### Phase 1: Make Archive Compile
1. **Fix SyntaxError in tools/patch_manager.py**
   - Line 157: Change `return "")` to `return ""`
   - This is blocking all compilation

2. **Fix tools package imports**
   - tools/test_runner.py and tools/log_parser.py incorrectly import from `.schemas` and `.tool_runner`
   - These should import from `agent_core.schemas` and `agent_core.tool_runner`
   - OR move shared schemas/tool_runner to tools/ directory

3. **Fix tools/repo_extractor.py field typo**
   - Dataclass field: `mime_type: Optional[str] = None`
   - __post_init__ incorrectly sets `self.mimetype` instead of `self.mime_type`

### Phase 2: Repair Package Structure
4. **Fix pyproject.toml packaging**
   - Current: `include = ["rfsn_v10*"]`
   - Should be: `include = ["rfsn_v10*", "agent_core*", "tools*"]`
   - Or remove agent_core/tools from release if not intended to ship

### Phase 3: Fix Agent Core Integration
5. **Fix test_agent_core_integration.py failures**
   - Orchestrator missing `add_test_result()` method
   - FinalOutput missing `success` field asserted in test
   - Need to add proper helper methods to Orchestrator

6. **Fix ToolRunner NameError**
   - In tool_runner.py: `stdout=strip_to_reasonable_length(stdout)` should be `stdout=self._strip_to_reasonable_length(stdout)`
   - Same for stderr

7. **Fix ToolRunner command parsing**
   - Replace `command.split()` with `shlex.split(command)` to handle quoted commands properly

8. **Fix permission matching order**
   - Sort rules by specificity or check deny/confirmation before broad allow rules
   - Prevent broad commands (pip, git) from overriding specific ones (pip install, git push)

### Phase 4: Fix RFSN Runtime Issues
9. **Add strict 4D validation in runtime.py**
   - Remove 3D support fallback
   - Explicitly validate: `len(keys.shape) == 4` and `len(values.shape) == 4`

10. **Add execution_mode to runtime returned info**
    - Telemetry event stores execution_mode but returned info dict omits it
    - Add: `"execution_mode": execution_mode`

11. **Avoid duplicate dense fallback computation**
    - When attention returns dense output (dense_requested, etc.), don't recompute
    - Only recompute if sparse path actually threw exception

12. **Add use_compressed_on_miss option**
    - On first cache miss, optionally store then retrieve to test compressed KV
    - Currently first call uses original KV, only later hits use compressed

13. **Split quantization and sparse audits**
    - Current audit only measures sparse-selection error on same KV
    - Need two audits:
      - quantized_dense_vs_original_dense (quantization error)
      - sparse_quantized_vs_quantized_dense (sparsity error)

### Phase 5: Fix KV Manager Issues
14. **Replace global RNG sign generation**
    - Current: `mx.random.seed(seed)` mutates global state (unsafe)
    - Fix: Use deterministic hash signs: `sign[i] = hash(i, seed) → +1 or -1`

15. **Add constructor validation**
    - Validate: `2 <= k_bits <= 8`, `2 <= v_bits <= 8`, `group_size > 0`

16. **Add group_size validation in retrieve()**
    - Currently only checks k_bits/v_bits mismatch
    - Add: `if cache.group_size != self.group_size: raise ValueError(...)`

17. **Set last_used on store**
    - New cache inserted with last_used=0.0 (ancient for LRU)
    - Fix: Set last_used=time.monotonic() during store()

18. **Apply transforms on shaped tensors**
    - Current: Flatten then apply sign/WHT (fragile)
    - Better: Apply transforms to shaped [B,H,T,D] tensors, then flatten

### Phase 6: Fix Telemetry Issues
19. **Fix ClickHouse client**
    - Add missing Optional import: `from typing import List, Dict, Any, Optional`
    - Fix malformed SQL: `values = ", ".join(lines)` not `"), (".join(lines)`
    - Add execution_mode to table schema
    - Consider switching to JSONEachRow for safer inserts
    - Route events to appropriate tables (not all to attention)

20. **Fix async_writer.py shutdown**
    - Current: stop() sets _stop_event before draining queue
    - Fix: Use sentinel-only shutdown, flush batch, queue.join(), then close

### Phase 7: Clean Up
21. **Remove generated files from release**
    - Delete __pycache__/ and *.pyc files from source archive
    - These should not be in release

22. **Update README accuracy**
    - Remove overclaims: production-grade, 198/198 tests passing, fused kernel
    - Accurate status: Alpha prototype, RFSN core improving, MLX tests required
    - Clarify agent core scaffold status, tools package state, telemetry limitations
